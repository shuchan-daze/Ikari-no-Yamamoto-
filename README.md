# 日報チェック by 怒りの山本

日々頭を悩ませる日報確認を、
できるだけ短時間で終わらせるために開発した確認支援アプリです。

OCRを使っていますが、
「AIが完璧に全部読み取る」思想ではありません。

実際の現場では、

- レシートが曲がる
- 光が反射する
- 手書きが雑
- 印字が薄い
- 撮影角度がズレる

こういったことが普通に起きます。

そのため、
OCRを盲信するのではなく、

「読めた候補を人間が短時間で確認・修正する」

という思想で設計しています。

---

# このアプリの特徴

- 日報撮影
- 明細撮影
- 過去写真選択対応
- OCRによる数字候補抽出
- 日報と明細の照合
- 差額確認
- 現金 / 未収分類
- 税抜営業収入計算
- 手動修正UI

---

# 開発思想

このアプリは、
「ChatGPTに頼めば誰でも簡単に作れる」
という種類のものではありません。

実際には、

- 現場業務理解
- UI設計
- OCR特性理解
- GitHub運用
- JavaScript実装
- iPhoneブラウザ挙動理解
- 業務フロー分析

こういった知識と経験が必要です。

特に重要なのは、
「現場で本当に使えるか」
を判断できる経験です。

単にコードを書くだけでは、
現場で使えるアプリにはなりません。

例えば、

- ボタン配置
- 撮影導線
- 再撮影誘導
- 修正のしやすさ
- 疲れた状態でも使えるUI

こういった部分は、
実際の業務経験がないと設計できません。

---

# なぜ作ったのか

毎日の日報確認は、
単純作業に見えてかなり神経を使います。

特に、

- 売上数字
- 未収
- 現金
- 伝票
- 貸切

などを確認していると、
人間なので必ず疲れます。

だからこそ、

「AIに全部やらせる」

ではなく、

「人間がラクになる」

方向へ設計しました。
